A (very) Brief Introduction to Git and Github

Why should I use Git/Github?

  • If you need a system to manage your software project (e.g. add/subtract features in an organized fashion), Git is one of the best/most popular version control systems
  • If you need to collaborate with other developers on software projects, Git is a way to handle this
  • If you need to publicize your project and attract contributors or users, Github can help with this

What is Git/Github?

Git: a program that keeps track of the files in a folder as you change them. It does this by storing snapshots of the state of folder. These snapshots are called commits.

Github: a "web-based Git repository hosting service". It's a place where you can store your code, and it offers many other features aimed at facilitating software development and collaboration.

Installing Git and Creating a Github Account

  • If you’re on Windows, you can install Git by going here. This option will also install an emulator for the Linux command line, which I will be using to interact with Git. Alternatively, you can use the Windows command line, or one of the many GUI’s out there.
  • If you’re on Linux or Mac, you should have Git already installed, or be able to install it through your package manager or other methods
  • Create a Github account by going to github.com and signing up

A Brief Tutorial

  • Git/Github is a big subject. I'll only attempt to introduce a few basic commands.
  • The plan: 1. Initialize a repository. 2. Push the repository to Github. 3. Make changes to both the local repository and the one on Github. 4. Pull the changes from the Github repository into the local one.

Step 1: Initialize a Repository

Running git init in your project folder creates another folder named ".git". This folder contains all the tracking information for your project.

Mark@LAPTOP ~/Desktop/Emerging_Tech_Website
$ git init
Initialized empty Git repository in C:/Users/Mark/Desktop/Emerging_Tech_Website/.git/
Mark@LAPTOP ~/Desktop/Emerging_Tech_Website (master)
$

Step 2: Check the status of the repository

  • Running git status gives you a summary of the repository's state, including the current branch, the current commit, and listings of uncommitted files that are either in the staging environment or untracked altogether.
  • The staging environment is where you can find files that are tracked by Git but not committed. The staging environment can seem unnecessary, but it's useful in certain scenarios, which I won't go into. 
Mark@LAPTOP ~/Desktop/Emerging_Tech_Website
$ git status
On branch master
Initial commit

Untracked files:
  (use "git add ..." to include in what will be committed)

        css/
        fonts/
        img/
        index.html
        js/

nothing added to commit but untracked files present (use "git add" to track)

Step 3: Make the first commit

  1. Run git add css fonts img index.html js to add all files to the staging environment.
  2. Run git commit -m "Add partially-complete files for website" to commit the files and associate a message with the commit.

Step 4: Create a repository on Github

  1. Log in to Github and click the "New" button on your Respositories page.
  2. Enter the appropriate information for an existing, public repository. You should now have an empty repository on Github.
  3. Run the following commands:
    4. git remote add origin https://github.com/markpratt/Emerging_Tech_Website.git
    git push -u origin master

    The first command marks the given URL as a remote repository and gives it the alias "origin". The second command pushes the commits from the local branch named "master" to the remote.
